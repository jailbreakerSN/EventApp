# PWA icons

Current state (staging hotfix, post-PR #153): the `manifest.webmanifest`
at the app root points at the **legacy** icons shipped alongside the
rest of the brand assets:

- `/icon-192.png` — 192×192, any purpose, generated by
  `scripts/generate-logo-assets.mjs`.
- `/icon-512.png` — 512×512, any purpose, same pipeline.
- `/apple-icon.png` — 180×180, iOS Safari home-screen icon.

This is sufficient for a working PWA install on Android Chrome and
iOS 16.4+. Future polish (deferred):

- `pwa-maskable-512.png` — 512×512, **maskable** (80% safe area per
  [maskable spec](https://web.dev/maskable-icon/)). Android
  launchers apply their theme's icon shape (circle, squircle, etc.)
  and without a maskable variant the icon gets letterboxed against
  a neutral background. Generate via an extension to
  `generate-logo-assets.mjs` with a 20% internal padding around the
  `teranga_event_logo_only.svg` icon.

When the maskable asset ships, add it to `manifest.webmanifest`
alongside the `any` entries — keep both, the browser picks the
right one based on context.
