## Authoring rules

1. **New documentation goes in `docs-v2/`.** Use the Diátaxis bucket that fits (`00-getting-started`, `10-product`, `20-architecture`, `30-api`, `40-clients`, `50-operations`, `60-contributing`, `70-future`, `99-reference`).
2. **Edits to 🟢 Canonical files in this tree are fine** — they haven't migrated yet, but they are authoritative until they do.
3. **Never edit a 🟡 Living audit in place.** Audits are point-in-time. Supersede with a new dated file and update `docs-v2/99-reference/audits.md`.
4. **Never edit 🔴 Archived files.** They exist for historical reference only.
5. **When migrating a 🟢 file to `docs-v2/`**, leave a one-line redirect stub here that points to the new location, then add a row to `docs-v2/99-reference/audits.md` (if it's an audit) or the relevant index.
