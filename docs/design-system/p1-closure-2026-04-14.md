## 4. Policy & tooling changes during P1

These landed alongside P1 because they unblocked the delivery model.

- **Target-develop branching policy** (PR #30 + `implementation-plan-2026-04-13.md` §1). Feature branches are still cut from the previous feature branch for WIP continuity, but PRs target `develop` directly for a flat review surface.
- **Auto-rebase workflow** (`.github/workflows/auto-rebase.yml`). Watches `push` on `develop` and force-pushes every open `feature/*` / `fix/*` / `refactor/*` / `chore/*` / `docs/*` branch onto the new tip via `peter-evans/rebase@v3`. Tested across 6+ cascades during P1; duplicate commits dropped via git's patch-equivalence detection.
- **Skills & subagents exercised** (per `.claude/skills/README.md`): `teranga-design-review` adapter + `ui-ux-pro-max` (rules 12, 29, 41, 55, 58, 62, 78) + `frontend-design` (form-field "never leave the user guessing") + `theme-factory` (dark-mode tokens) + `webapp-testing` (Playwright verifier pattern). Subagents: `@l10n-auditor` (I1 regressions), `@security-reviewer` (H6 + follow-up), `@firestore-transaction-auditor` (n/a — no service mutations touched).
- **Pre-commit agent discipline** documented in `implementation-plan §3` (agent × task matrix).

## 5. Deferred / known residuals (Wave 10 polish)

None of these block P1 closure. All are recorded here for the next sprint.

### 5.1 Security follow-up (open)

- **PR #38** — API-side `requireEmailVerified` middleware. Already opened with two rounds of `@security-reviewer` feedback addressed (subscriptions.routes.ts + receipts.routes.ts added, middlewares/index.ts barrel updated). Merge unblocks closing H6 entirely.

### 5.2 i18n long-tail (naturally absorbed)

- **Mechanical `t("…")` string swaps** in the files patched by `/tmp/bulk_i18n_patch*.py`. The hook is wired in each file (`const _t = useTranslations("common"); void _t;`) — incremental swaps flow naturally as future PRs touch those pages. No UX regression meanwhile (placeholder is a `void`).
- **Wolof coverage past the seed**. `wo.json` ships a curated seed per app; missing keys fall through to French via the runtime fallback in `i18n/request.ts`.
- **6 components the bulk patcher couldn't auto-match**: `event-card`, `share-buttons`, `event-detail-tabs`, `topbar`, `command-palette`, `UpgradeBanner`. Each uses a named-function or `forwardRef` pattern the regex didn't recognise. Manual touches in their next feature-touch PR.
