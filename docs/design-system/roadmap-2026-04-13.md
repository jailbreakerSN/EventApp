The `frontend-design` skill advises against Inter as a body font. **We override that recommendation** ‚Äî Inter ships in our design tokens, is Latin-complete for French/Wolof transliteration, and has proven accessible contrast at 14‚Äì16 px. The `teranga-design-review` adapter hard-codes this exception.

## 6. Verification plan

Driven by the `webapp-testing` skill once the CI/emulator environment is live. Scenarios:

1. **Sidebar responsiveness** ‚Äî 375 / 768 / 1280 px; hamburger opens drawer, Escape closes, backdrop dismisses, focus returns to trigger.
2. **Keyboard-only flow** ‚Äî Tab from `Aller au contenu principal` ‚Üí main; no trap outside modals; focus ring always visible.
3. **Modal focus trap** ‚Äî open any `<Dialog>`: Tab cycles within, Escape closes, focus returns to opener; background `aria-hidden="true"`.
4. **Forgot-password happy path** ‚Äî submit email ‚Üí Firebase Auth emulator log shows `sendPasswordResetEmail` ‚Üí success screen.
5. **Payment polling** ‚Äî simulate paid registration ‚Üí React-Query `refetchInterval` hits every 3 s ‚Üí terminal state rendered ‚Üí auto-redirect to badge (3 s countdown visible).
6. **Reduced motion** ‚Äî toggle OS `prefers-reduced-motion: reduce` ‚Üí Spinner, Skeleton, Toast, sidebar drawer all respect it (new global rule).
7. **Font swap** ‚Äî throttle to 3G, reload ‚Üí layout stable via fallback font, swap on load ‚Üí no FOIT.

Results will be appended to ¬ß7 once an automated harness is wired.
