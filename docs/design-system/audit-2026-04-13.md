## 3. Skills methodology

This audit was produced through the **`teranga-design-review`** adapter skill (`.claude/skills/teranga-design-review/SKILL.md`), which orchestrates four upstream skills while enforcing Teranga's locked brand tokens (teranga-navy / gold / green, Inter, French-first, XOF, WCAG 2.1 AA).

| Skill              | Role in this audit                                                                                       | Representative rule citations                                               |
| ------------------ | -------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------- |
| `frontend-design`  | Design-thinking guardrails on form affordances, empty states, and error states.                          | *"never leave the user guessing"* (→ H3 onBlur validation).                 |
| `theme-factory`    | Locked to Teranga palette; used only to confirm dark-mode tokens propagate through Badge call-sites.     | Dark-mode contrast ≥ 4.5:1 (→ H7 Badge call-site audit).                    |
| `webapp-testing`   | Playwright-script pattern for §7 verification scripts (`scripts/design-verification/`).                  | `with_server.py` lifecycle, headless chromium, `wait_for_load_state`.       |
| `ui-ux-pro-max`    | Challenger/reviewer; primary source of numbered rule citations below.                                    | rule 12 (discovery chips), rule 29 (snap-scroll), rule 41 (perceived perf), rule 44 (African-network perf), rule 55 (toast placement), rule 62 (empty-state quality), rule 78 (chart a11y), rule 83 (motion a11y). |

**Inter-override exception:** `frontend-design` advises against Inter for body copy. We override that — Inter is Latin-complete for French/Wolof, has proven contrast at 14–16 px in our context, and is locked in `design-tokens.md`. The adapter skill hard-codes this exception.

**Agents run against the `main` HEAD (commit `dd02c3d`):** `l10n-auditor` (see §5). `security-reviewer` was **not** invoked — P1 is UI-only, and the single enforcement item (H6) is scoped inline in the execution plan.

## 4. Findings

Grouped by category. Every finding has a file-path citation. Severity legend: **HIGH** = ship in the next branch; **MEDIUM** = next 2 branches; **NEW** = surfaced by skills, not in 04-07 audit.

### 4.1 Discovery & Navigation

| ID | Severity | Skill rule | Finding & evidence |
| -- | -------- | ---------- | ------------------ |
| **H1** | HIGH | `ui-ux-pro-max` rule 12 (discovery patterns — chips > selects on mobile) | `apps/web-participant/src/components/event-filters.tsx:118-166` uses **six `<Select>` dropdowns** (date, city, category, format, price, plus search). Price is binary free/paid in the select, not segmented. Chips would outperform dropdowns on `< 640 px`, the dominant viewport for the Senegal market. |
| **H2** | HIGH | `ui-ux-pro-max` rule 29 + a11y rule 78 | `apps/web-participant/src/app/(public)/events/[slug]/page.tsx` is a **single-page vertical scroll** — no tab strip exists at all. The 04-13 roadmap underscoped this as "overflow on mobile" — it is in fact a full greenfield implementation: introduce `<Tabs>` (already in `shared-ui`) for À propos / Intervenants / Sessions / Sponsors / Pratique, with URL-persisted state. |

## 5. l10n-auditor findings

Run date: **2026-04-13**. Agent: `.claude/agents/l10n-auditor.md`. Scope: `apps/web-backoffice/src/`, `apps/web-participant/src/`, `packages/shared-ui/src/`.

**Headline numbers:**

- **83 files** with hardcoded user-facing strings
- **~547 individual instances** across JSX text nodes, `placeholder=`/`aria-label=`/`title=`/`alt=` attributes, and `toast.*()` calls
- **Zero** components call `useTranslations()` or `getTranslations()` — `next-intl` is wired in both `next.config.ts` files but never invoked
- **Translation files are skeletal:** `src/i18n/messages/fr.json` (backoffice: 71 keys, participant: 33 keys), `en.json` present but mirror-skeletal, **no `wo.json` anywhere**

### Top 10 offenders (combined instance count)

| Rank | Instances | File                                                                                  |
| ---- | --------- | ------------------------------------------------------------------------------------- |
| 1    | 95        | `apps/web-backoffice/src/app/(dashboard)/events/[eventId]/page.tsx`                  |
| 2    | 36        | `apps/web-backoffice/src/app/(dashboard)/events/new/page.tsx`                        |
| 3    | 21        | `apps/web-backoffice/src/app/(dashboard)/settings/page.tsx`                          |
| 4    | 21        | `apps/web-backoffice/src/app/(dashboard)/organization/page.tsx`                      |
| 5    | 16        | `apps/web-backoffice/src/app/(dashboard)/admin/organizations/page.tsx`               |
| 6    | 15        | `apps/web-backoffice/src/app/(dashboard)/events/page.tsx`                            |
| 7    | 14        | `apps/web-participant/src/app/(authenticated)/sponsor/[eventId]/page.tsx`            |
| 8    | 14        | `apps/web-participant/src/app/(authenticated)/speaker/[eventId]/page.tsx`            |
| 9    | 14        | `apps/web-backoffice/src/app/(dashboard)/admin/venues/page.tsx`                      |
| 10   | 13        | `apps/web-backoffice/src/app/(dashboard)/badges/page.tsx`                            |

**By app:** web-backoffice: 35 files; web-participant: 42 files; shared-ui: 6 files.

### Top 5 string-pattern categories

1. **Toast feedback messages** — the densest category. Every `toast.success()`/`toast.error()` uses raw French (`"Événement publié."`, `"Inscription annulée."`). `(dashboard)/events/[eventId]/page.tsx` alone has 22.
2. **Form labels & section headings** — `<label>`, `<h3>`, `<dt>` in multi-step forms use literals (`"Titre"`, `"Nombre max de participants"`, `"Approbation requise"`).
3. **Table headers & status option labels** — `<th>`/`<option>` with French status vocabulary (`"Confirmé"`, `"En attente"`, `"Paiement en attente"`, `"Remboursé"`).
4. **Empty-state text** — `<p>` fallbacks (`"Aucune session programmée"`, `"Aucune publication"`, `"Aucun intervenant"`) hardcoded across event detail/list pages.
5. **Shared-UI component `aria-label`** — `pagination.tsx`, `dialog.tsx`, `search-input.tsx`, `file-upload.tsx`, `toaster.tsx` carry hardcoded French aria-labels. **Highest leverage** — every consumer inherits the violation.

### Remediation note

Infrastructure is already in place (`next-intl` installed and configured). The work is **mechanical extraction** — estimated 3–5 developer-days for the ~547 instances, plus a separate sourcing effort for Wolof translations. Framed as **TASK-P1-I1** in `execution-plan-2026-04-13.md`, with sub-tasks (shared-ui first — highest leverage; then top-10 offenders; then the long tail).
