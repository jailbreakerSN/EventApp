# Create Your First Event — End-to-End Walkthrough

> **Status: shipped** — This walkthrough follows the actual back-office UI and API as implemented.

This tutorial walks you through the complete life of an event: creation → ticket setup → publish → registration → check-in → badge download. It uses the local emulator stack from [Local Setup](./01-local-setup.md).

---

## Before you start

- Emulators running, seed data loaded (`npm run seed:emulators` or `npx tsx scripts/seed-emulators.ts`)
- Back-office running at http://localhost:3001
- Logged in as `admin@teranga.dev` (Pro plan — all features unlocked)

---

## Step 1 — Create the event

1. Click **Events** in the left sidebar.
2. Click **+ Nouvel événement**.
3. Fill in the details form:
   - **Titre** — `Hackathon Dakar 2026`
   - **Catégorie** — `conference`
   - **Format** — `in_person`
   - **Lieu** — Pick or type `Dakar`
   - **Date de début** — any future date
   - **Date de fin** — end date
4. Click **Créer l'événement**.

The event is created with `status: draft`. It is not yet visible to participants.

---

## Step 2 — Add a ticket type

On the event detail page, scroll to **Billets** (Tickets):

1. Click **+ Ajouter un billet**.
2. Fill in:
   - **Nom** — `Entrée générale`
   - **Prix** — `0` (free) or e.g. `5000` (XOF 5 000)
   - **Capacité** — `100`
3. Save.

> Free plan restriction: paid tickets (`price > 0`) require the `paidTickets` feature flag, which is only enabled on Starter+. On the Free plan the form rejects any price > 0 with a plan-limit error.

---

## Step 3 — Publish

Click **Publier l'événement**. The event `status` changes to `published`. It now appears on:

- The participant web app event discovery page (http://localhost:3002/events)
- The public API `GET /v1/events` endpoint

---

## Step 4 — Register a participant

Open a new browser tab (or an incognito window) and go to http://localhost:3002.

1. Find **Hackathon Dakar 2026** in the event list.
2. Click the event → **S'inscrire**.
3. Sign in with `participant@teranga.dev` / `teranga2026!` (or create a new account).
4. Select the ticket type and complete registration.

The participant receives a confirmation. A badge document is created asynchronously by the Cloud Functions trigger.

---

## Step 5 — View the registration in the back-office

Back in the back-office, go to your event → **Inscriptions** tab. You should see the participant listed with status `confirmed`.

From this view you can:
- Approve or reject pending registrations
- Export the participant list to CSV (Pro plan)
- Promote a participant to `staff` or `speaker`

---

## Step 6 — Check in the participant (live QR scan)

Go to your event → **Check-in** tab.

1. You will see a QR code input field (or camera scan on mobile).
2. In the back-office UI, type or paste the `qrCodeValue` from the participant's registration (visible in Firestore emulator UI at http://localhost:4000 → registrations collection).
3. Click **Scanner**.

A green confirmation appears with the participant's name. The registration `status` changes to `checked_in`.

> On mobile (staff app), the scanner uses the device camera. The QR payload is verified against the HMAC-SHA256 v4 signature; forged or expired QRs are rejected.

---

## Step 7 — Download the badge

The participant can download their badge from:

- **Participant web app**: http://localhost:3002/my-events → click the event → **Mon badge**
- **Back-office**: Event → **Badges** tab → download icon

The badge is a PDF generated by a Cloud Functions trigger (`onRegistrationCreated`) when the registration is first confirmed. If the PDF is not yet available, the badge status shows `pending` — wait a moment and refresh.

---

## What just happened under the hood

```
Participant registers  →  POST /v1/registrations
  └── RegistrationService.register()
        ├── checkPlanLimit(org, 'participantsPerEvent', count)
        ├── generateQrCodeValue(v4, HKDF key)
        ├── Firestore write (registration doc)
        └── eventBus.emit('registration.created')
              └── onRegistrationCreated (Cloud Function)
                    └── Creates badge doc (status: pending)
                          └── onBadgeCreated (Cloud Function)
                                ├── Renders PDF
                                ├── Uploads to Cloud Storage
                                └── badge.status = 'generated'

Staff scans QR  →  POST /v1/events/:id/checkin
  └── CheckinService.checkin()
        ├── Verifies HMAC-SHA256 signature (v4/HKDF)
        ├── Checks notBefore / notAfter window
        ├── Enforces scan policy (single / multi_day / multi_zone)
        ├── Checks access zone capacity
        ├── Updates registration.checkedInAt + checkedInBy
        └── eventBus.emit('checkin.completed')
```
